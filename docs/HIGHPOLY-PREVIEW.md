# High-Poly Preview for BF6 Portal Level Building

Build Portal maps against the **actual in-game models** instead of the SDK's
decimated grey proxies. This system overlays extracted, fully-textured
high-poly versions of every placed prop directly in the Godot editor — while
keeping the low-poly proxies as the untouched source of truth that exports to
the game.

![concept] Low-poly proxy (saved, exported) ⇄ High-poly textured overlay (editor-only)

---

## 1. Design contract

- **The low-poly proxy is the source of truth.** It is what gets saved in the
  `.tscn` and what the Portal exporter ships. Nothing in this system ever
  modifies it.
- **The high-poly is a disposable overlay.** Each swapped piece gets an
  `owner = null` child node named `_HIPOLY_PREVIEW` — Godot never serializes
  it, so it cannot leak into your scene file or your export. Reloading the
  scene silently drops all overlays; one click brings them back.
- **Toggle any time.** The High-Poly dock's Detail Mode dropdown drives the
  whole scene; `Selected → Current Mode` / `Selected → Low-Poly` work per node.

## 2. Installation (for a fresh SDK project)

1. Copy `addons/highpoly_toggle/` into the project's `addons/` folder.
2. Enable it: Project → Project Settings → Plugins → "BF6 High-Poly Preview".
3. That's it for normal use — the dock downloads models and map data on demand
   (see the repository README). The rest of this document covers the
   internals and the maintainer-side asset pipeline.

## 3. Deploying assets for a level (maintainer / pipeline path)

Normally users just click download in the dock. For pipeline work, one command
generates everything a specific level needs:

```
python tools/deploy_scene.py <path-to-level.tscn> [godot-project-dir]
```

What it does per placed prop:
1. Reads every `objects/<category>/<Name>.tscn` reference in the level file.
2. Resolves the proxy name to its extracted game mesh via `data/matches.tsv`.
3. If that mesh was never extracted, pulls it from the game dump on the spot
   (mesh extraction, shadow-submesh removal, texture decode).
4. Writes `res://highpoly/<Name>/<Name>.glb` — full PBR (base color, normal
   map, roughness), smooth vertex normals baked in (Godot needs them to
   generate tangents; without them every normal-mapped mesh warns).

Multi-part wrecks (e.g. `WreckTank_Abra01`) are defined in the script's
`ASSEMBLIES` table and merged from their parts — the game stores them in
shared assembly space, so the parts recombine exactly.

## 4. How the plugin matches and places overlays

Source: `addons/highpoly_toggle/highpoly_lib.gd`

**Matching** is by each instance's `scene_file_path` (the proxy scene it was
instanced from), so duplicates and renamed nodes all match; node-name matching
with trailing-digit stripping is the fallback.

**Placement** is conservative — the proxy's transform is trusted:
- The overlay inherits the proxy's transform untouched whenever its bounding
  box already agrees (ratio spread ≤ 1.35, scale within ±10%). Most props hit
  this path: pure identity.
- If the shape agrees but the size is off (the matched mesh is a different
  size variant), a **uniform** corrective scale is applied — never non-uniform.
- If identity fails, the six axis permutations are tried; a rotation is only
  applied when identity truly fails *and* the rotation truly fits.
- After any correction the overlay is re-centered onto the proxy's AABB;
  thin pieces (floors, panels) align their **top** surfaces so walkable
  surfaces stay flush.
- If nothing fits, the overlay is rejected and the proxy stays visible —
  a wrong-looking model is worse than a low-poly one.
- Props listed in `NOFIT` (verified pivot-aligned assemblies whose AABBs
  legitimately disagree, like the tank wreck) skip fitting entirely.

**Visibility** swaps hide every `GeometryInstance3D` under the proxy while
skipping overlay subtrees at any depth — nested matched pieces can't hide
each other's overlays.

**SDK validator compatibility:** the Portal SDK's level validator flags any
node instanced from a raw `.glb`. Overlays clear their `scene_file_path`
immediately after instantiation, so the validator ignores them. (They were
already unexported via `owner = null`; this just silences the alert.)

## 5. The matching database

`data/matches.tsv` maps all ~9,100 SDK proxy names to extracted game meshes
(columns: `godot_proxy, game_mesh, score, tier`). It was generated by
normalized name matching (89% coverage) and then **hand-corrected**; corrected
rows have `tier=manual` and must not be regenerated over.

Known failure modes of the automatic matcher (all instances in the reference
scene have been fixed, but expect these on new maps):
- **Destroyed-twin swaps** — proxy `Crate_03_A` matched to `dc_mil_crate_03`
  (the shot-up version) instead of `mil_crate_03`. `dc_` = destruction variant.
- **Size-variant swaps** — `WallFenceO01_A` (4.97m) matched to the 7.05m mesh
  while `_b` (4.98m) was the true twin.
- **Part-for-whole** — `WreckTank_Abra01` matched to just the barrel.

### Fixing a mismatch (the playbook)

1. **Check the model library site first** — search the prop's Portal name
   (site card titles come from the SDK's `RuntimeSpawn` enums, so the mapping
   is authoritative). This usually answers it instantly.
2. Compare candidate dimensions against the SDK proxy AABB
   (`data/proxy_aabb.tsv`); the true twin usually matches to the centimeter.
3. Run `tools/audit_scene.py` for a whole-scene sweep: it verifies every
   deployed prop's dims, auto-extracts missing family variants from the dump,
   auto-pins conclusive winners, and shortlists ambiguous ones for visual review.
4. Pin the fix in `matches.tsv` (`tier=manual`) and re-run `deploy_scene.py`.

## 6. How the textures are made (summary)

The GLB textures come from a static extraction pipeline that reproduces the
game's Frostbite material compositing. Key facts (full history in
`DISCOVERIES.md`):

- **Base color / normal**: decoded from the prop's co-located `.Texture`
  resources; each texture's header (~byte 128) declares its role
  (`World_BaseColor`, `World_Normal`, `World_Vista`...). BCn `*_sRGB` DXGI
  formats need remapping to their UNORM twins before decoding.
- **Shadow submeshes**: every MeshSet also contains shadow/depth-only
  submeshes (`M_*_Shadow`, `*_ZOnly`) with no UVs; they must be dropped or
  they wrap the prop in a black shell.
- **Shared-material props** (plaster walls, facades): the base sheet is tiled
  by **UV0 divided by the material's tiling params** (two floats read from the
  mesh EBX via the reflection type-SDK), sampled within the sheet's uniform
  band; the per-object `vst_*_nsm` vista bake supplies the baked normal +
  smoothness; close-up grain comes from the shared
  `common/shaders/textures/prop/ta_propdetailarray.Texture` (8-slice
  `World_DetailArray`); everything is baked into the mesh's UV1 unwrap.
- **Limits**: a small set of props (emissive light bars, some vegetation)
  use texture layouts the pipeline doesn't cover yet — they stay low-poly.

## 7. Performance notes

- Overlays exist only in the editor; game exports are unaffected.
- GLB textures are capped at 1024²; a full scene of ~1,300 overlays is
  workable on a mid-range GPU. Use `Selected → High-Poly` to preview only the
  area you're detailing if the editor gets heavy.
- After deploying new GLBs, Godot needs a moment to import them; if a piece
  doesn't swap on the first click, hit **Re-apply Scene** again once the
  import finishes (the plugin skips assets that aren't imported yet rather
  than erroring).

## 8. File map

| Path | Role |
| --- | --- |
| `addons/highpoly_toggle/` | the editor plugin (this repo, canonical copy) |
| `<project>/addons/highpoly_toggle/` | installed copy |
| `<project>/highpoly/<Name>/<Name>.glb` | per-prop preview assets |
| `tools/deploy_scene.py` | one-command asset deployment for a level |
| `tools/audit_scene.py` | whole-scene match verification/repair |
| `data/matches.tsv` | proxy → game-mesh database (manual pins!) |
| `data/proxy_aabb.tsv` | SDK proxy dimensions (match verification) |
| `A:\bf6highpoly_textured\<mesh>\` | extracted OBJ + PBR textures per mesh |
| `site/` | browsable model library (Portal names ↔ game meshes) |
